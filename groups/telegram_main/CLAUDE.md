## Formatting

This chat is on Telegram. Tables are rendered as monospace code blocks with limited width (~45 chars before wrapping on mobile).

When outputting tables:
- Use short headers (e.g., "Hi/Lo" not "High/Low", "Wind" not "Wind Speed")
- Drop units from every row — put them in the header instead (e.g., "Rain(mm)" then just "17.4")
- Use 1-space column gaps, not 2
- Omit columns that add little value if the table is too wide
- Prefer compact date formats (e.g., "Fri 20" not "Fri 20 Mar")
- Skip emoji in table cells — they break monospace alignment
